# yay_music_bot — 小規模手動運用ガイド

方針（2026-06-06 究決定）: **Yay で小さく、知り合いに手動で提供**。垢量産なし、実アカ1〜2個、課金は後（究GO必須）。
このガイドは「友達1人に音楽botを使わせる」までの手順。

## 提供モデルは2つ

### Model A: bot が友達の通話に入る（`serve.sh`）
- 友達が自分の通話を開く → そこに bot が入る → 友達が `/p 曲名` で操作。
- **未検証の前提**: bot アカが「他人の通話」に入れるか。Yay の通話が follow/招待制だと、
  友達が bot アカをフォロー/招待する必要があるかもしれない。**最初の友達で必ず実証する**。

### Model B: 友達が bot の通話に来る（一番確実）
- bot を起動して通話に居させ、その通話に友達が入ってくる（共有の音楽部屋）。
- 「bot が他人通話に入れるか」問題を回避できる＝確実。みんな1部屋に集まる形。
- 究の個人 yay_bot と同じ「自分の通話に bot」経路なので、すでに動く実績あり。

→ まず **Model B で確実に提供**を始め、Model A は権限を実証できたら拡張、が安全。

## 友達の Yay user id の調べ方（Model A 用）
- 友達の Yay プロフィールを開く → URL/共有リンクの `/user/<数字>` の数字が user id。
- それを `serve.sh` の第1引数に渡す。

## 手順

### Model B（推奨・確実）
1. bot アカで Yay 通話を開く（究が手で、または bot アカ端末で）。
2. `./run.sh` で bot 起動（その通話を自動発見して参加）。
3. 友達にその通話へ来てもらう。
4. 友達がチャットで `/p 曲名` や「○○かけて」。`/h` でコマンド一覧。

### Model A（友達の通話に入れる場合）
1. 友達に通話を開いてもらう。友達の user id を聞く（上記の調べ方）。
2. `./serve.sh <友達のuid>` 実行 → bot がその通話に入る。
3. 入れなければ Yay の権限（フォロー/招待）が原因。友達に bot アカをフォロー/招待してもらって再試行。
4. 停止: `./serve.sh stop <友達のuid>`

## アカウントの考え方
- **1アカ = 1用途**。究の個人 `yay_bot` と音楽提供を同じアカで同時に動かすと SAME_UID で蹴り合う。
- 友達提供用に bot アカを1個分けるのが綺麗（実アカ。垢量産はしない）。
  分けたら relogin でそのアカのトークンを別ファイル（例 `.yay_token_bot2`）に取得し、
  `./serve.sh <uid> .yay_token_bot2 <そのアカのselfUid>` で使う。
  ※ relogin の別アカ対応（出力トークンファイル切替）は、2アカ目を用意する時に追加する。

## 課金（後回し・究GO必須）
- 小規模なら最初は手動でいい（個別に集金 or 無料で身内に）。Stripe 等の自動課金は § 課金ルールで究の明示GOが要る。
- 自動化するなら最小構成: Stripe Payment Link を1つ作る → 払った人にだけ提供。ダッシュボードは不要。

## トラブル
- bot が通話を見つけない: 対象アカがその通話に居るか（Model B は bot アカ自身、Model A は友達）を確認。`yay_api.py active <uid>` で確認。
- 音が出ない: `/v 30` 等で音量。`/np` で再生状態。
- トークン切れ: `relogin.sh`（Xログイン1クリック）。
