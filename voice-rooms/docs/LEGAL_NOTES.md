# voice-rooms — 法的ガードレール

「Yay! みたいなもの」を作るうえで訴訟リスクを避けるための恒久メモ。結論: **機能・レイアウト・概念の模倣は合法。名前・ロゴ・素材・文章・造語の流用と、外観の丸パクリが危険**。

## OK（守られない＝自由に真似てよい）
- アプリの概念（匿名ボイスSNS）、機能セット（通話ルーム/サークル/タイムライン/DM/通知）
- 情報設計・画面構成・ナビ構造（タイムライン＋下タブ＋カバー付きカード 等）。UI慣習は著作権・特許の対象外
- WebRTC 等の標準技術の利用

## NEVER（やると危険）
1. **商標**: 「Yay!」「ナナメウエ / nanameue」のロゴ・名称、造語「だれ通」を**自社名・サービス名・機能名・UIラベルに使わない**。
   - 自前の独自名・ロゴ・ブランドを用意する（現状コードネーム "voice rooms" は純記述的で商標リスク最小、ただしブランドとしては要・本命命名）。
2. **著作物コピー**: 相手の**コード・CSS・class名・アイコン・画像・商品文/UI文言**をそのまま流用しない。全て自前生成する（現状クリア＝オリジナル生成）。
3. **トレードドレス（外観丸パクリ）**: 配色まで同一にして出所混同を招かない。
   - 回避済み: Yay=白＋マゼンタ #e849be / 本作=黒 #0a0a0a＋ローズ #ff5e7e で別アイデンティティ。
4. **規約類**: 利用規約・プライバシーポリシーを相手から流用しない（出すなら自作）。
5. **シード/モック文の固有名**: 実在アーティスト名・実在ブランド・第三者作品名を本文に残さない。
   - 実施済みscrub: "William Basinski"→汎用、"だれ通"→「オープン通話」、served HTML 内の "Yay" 参照コメント→汎用語。

## 公開リポジトリ運用上の注意
- 本PJは `Emocute/sketches`（tracked）。**公開する場合**は `docs/ABSTRACT.md` 等の内部資料に残る "Yay" 比較記述が「Yayクローンです」という外形証拠にならないか一読する（研究/設計の参照としての言及自体は適法だが、印象管理として要点検）。
- 配布・公開前チェック（`Downloads/CLAUDE.md § 販売物リリース前監査` に準拠）:
  `grep -oiE "yay|nanameue|だれ通|basinski|<実在名>" public/index.html public/_screens/*.html` がゼロであること。

## 製品化するなら（将来）
1. 独自の**名前＋ロゴ＋ブランド**を確定（"Yay" を想起させない）。
2. 自作の**利用規約・プライバシーポリシー**（匿名・年齢確認・通報の運用方針を含む）。
3. 未成年保護・出会い目的禁止など、Yayが法的トラブルで学んだ論点（前身ひま部の児童被害）を**設計で先回り**（年齢帯ゲート・部屋内通報/kick は実装済みの骨子）。
4. 音声/録音の扱い、外部ID交換の抑止など、運用ポリシーを明文化。
