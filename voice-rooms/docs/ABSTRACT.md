# voice-rooms — アブストラクト & スケルトン

機能性最優先。ナラティブはフレーバー程度。Yay! の強みを移植し、弱みを潰す web-first 匿名通話。

## アブストラクト（一段落）

ブラウザだけで動く、匿名のオープン通話ルーム。アプリ不要・電話番号不要・ログイン不要で、開いている部屋に飛び込んで声で話せる。趣味タグで部屋を探し／立て、「今ひま」を出して即つながる。Yay! の中核（だれ通・サークル・ひまなう）を web に移し、Yay! が web で出来ない「ブラウザ通話」をそのまま強みにする。重さ・業者・治安という Yay! の三大不満を、軽量実装・部屋内モデレーション・一時性の設計で抑える。

## Yay! から盗む（強み移植）

| 盗む機能 | Yay! での姿 | 移植方針 |
|---|---|---|
| **だれ通** | 誰でも入れるオープン通話ルーム | 中核。ロビーの主役。匿名で即入室 |
| **サークル** | 趣味別コミュニティ（9万超） | まずは部屋に**趣味タグ**を付ける軽量版。永続コミュニティは後 |
| **ひまなう** | 「今ひま」状態でマッチ | 在席ステータス。出すと「話せる人」一覧に載る |
| **タイムライン** | X風の投稿 | 後回し（通話と非同期で軽く） |
| **同世代マッチ** | 年齢帯で接触制御 | 安全機能として後で。MVPは任意の年齢帯フィルタ |
| **連絡先交換不要の安心感** | 匿名・身バレなし | **これが情緒的価値の核**。連絡先要求を設計で抑止 |
| **プロフィール検索** | 趣味/年齢で検索 | タグ検索で代替 |
| **ブロック/追放/24h監視** | モデレーション | 部屋内 通報・ホストkick・ミュートで最小実装 |

## Yay! の弱み → 対策（差別化）

| Yay! の不満 | 出典傾向 | 我々の対策 |
|---|---|---|
| 重い・落ちる・電池が熱い | レビュー58%が不具合系 | **軽さ最優先**: 依存最小、メッシュ少人数、無駄な再描画なし。落ちない > 機能盛り |
| 業者・なりすまし・LINE誘導・詐欺 | 口コミ多数 | 外部ID誘導の検知/抑止、部屋内通報、新規レート制限。匿名は保つが「持続的な悪意」を持てない設計 |
| 下ネタ・通話強要・民度 | 口コミ多数 | ホスト権限（mute/kick）、部屋ノルマ表示、一時性で悪人が居つけない |
| 急に消える・関係続かない | 口コミ多数 | 一時性は受容。連絡先交換なしの「また会える」軽い再会導線（任意） |
| 通知が一瞬で消える | レビュー | 確実な通知（後フェーズ。MVPは在席で代替） |
| 広告でかい・VIPで機能制限 | レビュー | MVPは邪魔な広告ゼロ。基本機能に課金ゲートを置かない |
| **Web で通話不可**（1:1も だれ通も） | 公式仕様 | **最大の刃**: web-first WebRTC で通話が動く。インストール不要 |
| 電話番号必須・日本語前提・円課金のみ | 海外レビュー | 電話不要・言語非依存・通貨非依存 |
| 投稿コメント削除不可 | 海外レビュー | 自分の投稿/発言は消せる |

## 非機能要件（＝機能性重視の核）

1. **軽い**: 初期ロード最小、依存は ws のみ。通話は P2P。多人数は SFU 差し替え前提で設計を汚さない
2. **落ちない**: 接続状態を常に可視化、再接続を自動化、片切れを検知して掃除
3. **電池に優しい**: 不要な常時アニメ/ポーリングを避ける（発話検出は rAF で軽量に）
4. **インストール不要**: URL を開くだけ。共有しやすい

## 機能スケルトン（優先順）

```
[MVP / 今ここ]
  ロビー
    - 匿名アイデンティティ（ニックネーム任意・編集）
    - 趣味タグ（自分のタグ）
    - 「今ひま」トグル（在席）
    - 部屋を作る（名前＋タグ）
    - 開いてる部屋一覧（タグ・人数・発話中数・タグ絞り込み）
  通話ルーム
    - 参加者タイル（発話インジケータ・接続状態・ミュート状態・ホスト印）
    - 自分: ミュート / 手を挙げる / 退出
    - 部屋テキストチャット（軽量・流れるだけ）
    - 参加者: 通報（stub）/ ホストなら kick（stub）
    - ひとり時の在席表示

[次フェーズ]
  - 永続サークル（タグ→コミュニティ昇格、再訪）
  - 確実な通知 / 在席プッシュ
  - 年齢帯フィルタ・本人確認の土台
  - 「また会える」再会導線（連絡先交換なし）
  - SFU 化で 18人規模（Yay! のグループ上限）
  - タイムライン（非同期の薄いつながり）
```

## データ / 通信スケルトン

- **シグナリング/部屋管理**: WebSocket（`server.js`）。状態は in-memory（MVP）
- **音声**: WebRTC mesh（〜6人）。`stun:stun.l.google.com:19302`
- **メッセージ種別**:
  - `join {room, nick, tags}` → `peers` / `peer-joined` / `peer-left`
  - `signal {to, data}` 中継（SDP/ICE）
  - `room {kind:'chat'|'hand', text}` → 同室へ broadcast
  - `rooms` 一覧（name, count, tags, talking）
- **クライアント状態**: nick / tags / muted / handRaised / 在席 / 部屋

## スタイル（フレーバーのみ）

- Emocute 実パレット: 背景 `#0a0a0a`、アクセント ローズ `#ff5e7e`、文字は可読下限を割らない
- 絵文字ゼロ・SVG アイコン・単色背景・装飾なし
- **ただし文言は機能本位**。詩的ナラティブは置かない（v1「声だけ」は `public/_archive/` に退避）
