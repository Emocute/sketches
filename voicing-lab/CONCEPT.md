# Voicing Lab

ブラウザ単一 HTML で完結する「ボイシング実験室」。1 つのコード（ルート + クオリティ）を入れると、同じコードを **Close / Drop2 / Drop3 / Rootless / Quartal / Shell / Spread（4-5声テンション）** の各ボイシングで並べ、ピアノ鍵盤上に可視化し、その場で鳴らし比べる。

## 目的

Studio の `v7_runtime/emocute/voicing.py` が内部関数として持っている和声哲学を、誰でも耳で確かめられる toy にする。

- 「ルートレスはピアノ単独だと寂しい、stab 系なら効く」
- 「6声フルテンションは盛りすぎ、4-5声+適度なテンションがスイートスポット」
- 「drop2 で内声を開くと急に大人になる」

——こういう Studio の蓄積を、言葉でなく **音と図** で体感できる場所。

## 既存との被り回避

| 既存 | 形態 | こいつとの違い |
|---|---|---|
| HarmonyScope | VST3 和声「解析」 | あっちは検出表示、こっちは生成・鳴らし比べ |
| Harmonizer | VST3 進行「リハモ」 | あっちは進行を別進行へ、こっちは単一コードのボイシング展開 |
| Geomonic | VST3 幾何 MIDI「生成」 | あっちは DAW へ MIDI、こっちはブラウザ完結の教育 toy |

ボイシング単体を扱うツールは既存に存在しない（空白地帯）。

## 技術スタック

- 単一 `index.html`（依存は Tone.js CDN のみ、ビルド不要）
- 度数表記は **Studio v7 準拠**（メジャースケール度数、I=0/II=2/III=4/IV=5/V=7/VI=9/VII=11）
- コードインターバルは `chord_defs.py::CHORD_INTERVALS` 相当を JS にミラー（共有コードは切らない＝Sketches 規約 3）

## 最小スコープ（v1）

1. ルート選択（C–B）+ クオリティ選択（maj7/min7/7/m7b5/dim7/maj9/min9/9/13/m6/6/add9/7sus4/mMaj7/7alt）
2. 7 ボイシングタイプを縦並びで同時表示、各々ピアノ鍵盤にノートハイライト + テンション度数ラベル
3. クリックで単体再生 / 「順に鳴らす」で全比較
4. 文脈プリセット（ピアノ単独 / stab / comping）で「この文脈ならコレ」を強調表示
