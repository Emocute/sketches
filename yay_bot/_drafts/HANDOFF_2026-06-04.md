## 🚨 運用の落とし穴（今回ハマった）
- **同一アカ多重ログイン= `SAME_UID_LOGIN`/`UID_BANNED` で蹴られる**。原因= bot/Chromium 孤児プロセス。
  `tmux kill-session` だけだと Playwright Chromium が孤児化し publish継続→次起動と uid 衝突。今回 chromium 6個/bot 2個に増殖していた。
- **クリーン再起動手順（必須）**:
  ```sh
  pkill -9 -f bot_agora.mjs
  pkill -9 -f "ms-playwright.*[Cc]hromium"      # 孤児Chromium掃除（究の実Chromeは無事）
  sleep 2
  ./scripts/voicevox_engine.sh start            # 先にTTSエンジン
  .venv/bin/python yay_api.py check             # expired→ ./relogin.sh（X手動ログイン1回・自動入力NG）
  tmux kill-session -t yay_bot 2>/dev/null
  tmux new-session -d -s yay_bot "YAY_AUTO_LIVE=BlackHole YAY_MUSIC_VOL=15 node bot_agora.mjs 2>&1 | tee /tmp/yay_bot.log"
  ```
  起動後: `grep -E 'SAME_UID|UID_BANNED' /tmp/yay_bot.log` が空、`pgrep -f 'ms-playwright.*[Cc]hromium'|wc -l` が1。
- **既定**: 読み上げ/聞き取りOFF（究「基本オフ」）、Spotify配信(AUTO_LIVE)はON。
